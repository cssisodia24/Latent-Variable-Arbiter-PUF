Latent Variable Modeling for Arbiter PUFsCourse: CS771 - Machine LearningInstitution: Indian Institute of Technology (IIT) KanpurAuthor: Chandraveer Singh SisodiaThis repository contains an implementation of a Maximum Likelihood Estimation (MLE) framework to model a 17-bit Arbiter Physical Unclonable Function (PUF) that contains missing latent data.📖 OverviewThe primary objective of this project is to estimate the physical parameters (weights and biases) of a 17-bit Arbiter PUF from a given set of Challenge-Response Pairs (CRPs).A key complexity in this dataset is that the 9th bit of every challenge is latent (hidden). Because the objective function is non-convex with respect to simultaneously finding both the PUF parameters and the latent bits, this problem cannot be solved using standard linear techniques. Instead, it is formulated and solved using an Expectation-Maximization (Hard-EM) alternating optimization pipeline.⚙️ ImplementationsTwo different approaches were tested to handle the latent variables during the EM process:1. Unconstrained Prior (my_latent)Assumption: The latent variable is unconstrained, operating on a flat prior of $P = 0.5$.Mechanism: The M-step fits the 17-bit model, and the E-step updates the latent bits strictly to minimize the 17-bit prediction error.Result: The model heavily overfits the training dataset. It artificially updates the latent bits to force mathematical convergence, yielding a deceptive 99.99% training accuracy, but ultimately fails to capture the true underlying hardware parameters.2. Generative Prior Regularization (my_latent_updated)Assumption: The latent variable is not random; it is deterministically generated by a secondary, hidden 16-bit Arbiter PUF.Mechanism: The E-step is heavily constrained. It requires a latent bit update to maximize the joint probability of both the 16-bit hidden model and the final 17-bit response.Result: This dual-model constraint acts as a powerful structural regularizer. By restricting the latent updates to physically probable hardware states, the EM algorithm successfully converges on the true parameters, achieving a robust 98.40% test accuracy on unseen CRPs.🚀 Optimization DetailsTo handle the dataset efficiently and avoid Python-level bottlenecks, the implementation includes specific performance optimizations:Vectorization: Feature mapping ($\phi(c)$), margin calculations, and log-sigmoid evaluations are fully vectorized using NumPy. This allows the 8,000-sample dataset to be processed rapidly without standard for loops.Warm-Started Solvers: The M-step leverages sklearn.linear_model.LogisticRegression with the warm_start=True parameter. This retains the weight matrices across EM iterations, significantly minimizing solver overhead and accelerating convergence.📂 Repository Structuresubmit.py: The core Alternating Optimization (Hard-EM) implementation.public_trn.txt: The training dataset containing 8,000 CRPs.public_tst.txt: The testing dataset containing 2,000 unseen CRPs.💻 UsagePrerequisitesThis project requires Python 3 along with the following libraries:numpyscikit-learnExecutionEnsure that submit.py, public_trn.txt, and public_tst.txt are all placed in the same root directory.Run the script from your terminal:
